# RAG（检索增强生成）技术原理

## 什么是 RAG
RAG（Retrieval-Augmented Generation）是一种将信息检索与文本生成相结合的技术。它在 LLM 回答用户问题之前，先从知识库中检索相关文档片段，将其作为上下文提供给 LLM，从而生成更准确、更具事实依据的回答。

## 核心流程
1. **文档加载**：读取知识库中的文档（Markdown、PDF、Word 等）
2. **文本切分**：将长文档切成固定大小的块（chunk），每个块约 200-1000 tokens
3. **向量化**：使用 Embedding 模型将每个文本块转换为向量
4. **向量存储**：将向量存入向量数据库（如 ChromaDB、Pinecone、Weaviate）
5. **用户提问**：将用户的问题同样向量化
6. **相似度检索**：在向量库中查找与问题最相似的 K 个文本块
7. **上下文组装**：将检索到的文本块作为上下文，拼接到 prompt 中
8. **LLM 生成**：将 prompt 发送给 LLM，生成最终回答

## Chunk 策略
- **chunk_size**：每个块的大小（字符数或 token 数），通常 300-1000
- **chunk_overlap**：块之间的重叠部分，用于保持上下文连贯性
- **separators**：切分时的优先分隔符顺序

对于中文文档，建议按 `\n\n` → `\n` → `。` → `！` → `？` → `；` → `，` → ` ` → `` 的优先级切分，以保证语义完整性。

## Retrieval 策略
- **Top-K**：返回最相似的 K 个块。K 值太小可能信息不足，太大会超出 LLM 上下文窗口
- **相似度算法**：通常使用余弦相似度（cosine similarity）
- **MMR**（最大边际相关性）：在相关性和多样性之间取得平衡

## 常见的 RAG 优化
- **HyDE**：先让 LLM 生成假设回答，再用该回答检索
- **Self-Query**：让 LLM 分析问题中的条件（时间、类别等）并结构化检索
- **RAPTOR**：基于摘要的递归检索，适合处理层次化文档
